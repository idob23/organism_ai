# Organism AI — Roadmap

## Фаза 1: Sprints 1–8 (завершены)

Базовая платформа: CoreLoop, Planner, Evaluator, Memory (pgvector), 8 инструментов, 4 агента, MCP клиент/сервер, A2A протокол, self-improvement, benchmark (26 задач).

## Фаза 2: Sprint 9 — Agent Factory + Reliability

### Цель
Автоматическая декомпозиция больших задач + инфраструктура для создания команды AI-агентов. "Нажал кнопку — получил отдел маркетинга."

### Блок A: Автоматическая декомпозиция задач (приоритет 1)

- Q-9.1: Task Decomposer — если задача требует >7 шагов, автоматически разбивается на подзадачи и выполняется последовательно через оркестратор с передачей контекста. Пользователь получает финальный склеенный результат, ничего не замечая. Постоянное решение проблемы FIX-17.

### Блок B: Agent Factory (приоритет 2)

- Q-9.2: Шаблоны ролей агентов — готовые роли: маркетолог, аналитик, закупщик, юрист, HR. Каждая роль — набор инструментов, промптов, расписаний и PERSONALITY.md
- Q-9.3: Автогенерация PERSONALITY.md — пользователь описывает роль на русском → система генерирует конфигурацию агента за 30 секунд
- Q-9.4: Мета-оркестратор — распределение задач между агентами по специализации. Умная маршрутизация: аналитическая задача → агент-аналитик, юридический вопрос → агент-юрист
- Q-9.5: Команды управления командой — /agents (список), /create_agent <роль>, /assign <агент> <задача>

### Блок C: Технический долг (приоритет 2)

- Q-9.6: Multi-tenancy — artel_id на таблицах есть, включить фильтрацию. Готовность к нескольким клиентам одновременно
- Q-9.7: Docker production deployment — docker-compose для production: PostgreSQL + бот + мониторинг. Нужно для первого клиента (приоритет 1)
- Q-9.8: MCP протокол JSON-RPC 2.0 — сейчас упрощённый HTTP REST. Для совместимости с Cursor, Claude Desktop и внешними системами

### Блок D: UX (приоритет 3, выяснилось при тестировании)

- Q-9.9: Прогресс по подзадачам — когда задача разбита на части, пользователь видит "Часть 1/3 готова..." вместо молчания
- Q-9.10: /errors команда — просмотр последних WARNING-событий из Telegram без захода на сервер

### Порядок выполнения
1. Q-9.1 + Q-9.7 — первыми (декомпозиция и Docker для клиента)
2. Q-9.2 → Q-9.5 — Agent Factory
3. Q-9.6, Q-9.8 → Q-9.10 — долг и UX
