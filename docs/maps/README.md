# Карты архитектуры Organism AI

Reference artifact проекта. Описывают устройство платформы на
разных уровнях детализации. Обновляются при каждом архитектурном
изменении — как CLAUDE.md и ARCHITECTURE_DECISIONS.md.

## Как использовать

**При планировании изменений.** Перед написанием промпта для Claude
Code открыть соответствующую карту, убедиться, что понимаешь где
именно в системе находится изменяемый компонент.

**При отладке.** Начинай с request_flow — по шагам от входа до
ответа легче понять, где ломается. Для проблем с памятью —
memory_flow. Для странного качества оценок — self_improvement_loop.

**При онбординге.** Главная карта даёт обзор за 5 минут. Дальше
углубляешься по конкретной области.

## Правило обновления

При любом PR, меняющем архитектуру:
1. Определить, какие карты затронуты
2. Обновить SVG (или попросить помощи у senior architect chat)
3. Указать в отчёте коммита "Обновлены карты: X, Y"
4. Если ни одна карта не обновлена — явно сказать в коммите
   "Архитектурных изменений нет, карты не затронуты"

Невыполнение этого правила создаёт дрейф между реальным кодом и
reference artifact. Через полгода карты становятся бесполезными.

## Как смотреть

SVG рендерятся в GitHub при просмотре онлайн. Локально — открыть
в любом современном браузере (Chrome, Firefox, Safari). Для печати
на A1/A2 — открыть в браузере, Ctrl+P, выбрать формат бумаги.
