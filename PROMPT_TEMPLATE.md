# Шаблон промпта для Claude Code

> Основано на 9 спринтах опыта. Промпты с этими секциями работают с первого раза.
> Промпты без них — требуют итераций.

---

## Шаблон

**Задача [ID]: [Краткое название]**

=== КОНТЕКСТ ===
[Что уже есть. Какие файлы затрагиваются. Как это связано с остальной системой.]

=== ЗАДАЧА ===
[Что конкретно нужно сделать. Пронумерованные шаги.]

=== КОНТРАКТ ===
Вход: [что получает компонент]
Выход: [что должен вернуть]
Побочные эффекты: [что меняется — БД, файлы, другие компоненты]

=== ЗАВИСИМОСТИ ===
[Какие модули, интерфейсы, паттерны повторяются]

=== ЧЕГО НЕ ДЕЛАТЬ ===
[Явные запреты]

=== ПРОВЕРКА ===
[Как убедиться что задача выполнена]

=== ОТЧЁТ ===
После завершения выведи:
РЕЗУЛЬТАТ: [успех/частично/провал]
BENCHMARK: X/Y (было Z/Y)
КАЧЕСТВО: X.XX (было Y.YY)
ИЗМЕНЁННЫЕ ФАЙЛЫ: [список]
ПРОБЛЕМЫ: [если есть]

Обнови CLAUDE.md, ARCHITECTURE_DECISIONS.md, organism_ai_roadmap.md (если нужно).
Git commit с префиксом задачи.

## Когда какие секции обязательны

| Тип задачи | Обязательные секции |
|-------------|-------------------|
| Новый tool/компонент | Все |
| Баг-фикс | КОНТЕКСТ + ЗАДАЧА + ЧЕГО НЕ ДЕЛАТЬ + ПРОВЕРКА + ОТЧЁТ |
| Рефакторинг | КОНТЕКСТ + ЗАДАЧА + ЗАВИСИМОСТИ + ЧЕГО НЕ ДЕЛАТЬ + ПРОВЕРКА + ОТЧЁТ |
| Документация | ЗАДАЧА + ПРОВЕРКА + ОТЧЁТ |
| Быстрый фикс (<10 строк) | КОНТЕКСТ + ЗАДАЧА + ОТЧЁТ |

## Правила написания промптов

1. **Сначала поиск** — найти актуальный код всех затрагиваемых файлов.
   Без этого промпт будет неточным.

2. **Контекст зависимостей > описание кода** — Claude Code умеет писать код.
   Он не умеет угадывать, как компонент связан с остальной системой.

3. **Явные запреты > надежды** — если может сломать что-то смежное,
   скажи "НЕ трогай X" явно.

4. **Один промпт = одна задача** — не смешивать фикс бага и новую фичу.

5. **Блок ОТЧЁТ — всегда** — без него нет feedback loop.
